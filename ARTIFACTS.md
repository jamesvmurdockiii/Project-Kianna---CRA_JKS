# Artifact Policy

CRA separates source code, compact evidence, and raw/generated artifacts. This
matters because the repository is public and Apache-2.0: reviewers should see a
clean research artifact, not a private Downloads folder or a full hardware cache.

## Source Of Truth

These files are public source-of-truth material and should generally remain in
Git:

- `README.md`, `LICENSE`, `CONTRIBUTING.md`, `RUNBOOK.md`, `Makefile`, `pyproject.toml`
- `ARCHITECTURE.md`, `MICROCIRCUIT_DESIGN.md`, `CONTROLLED_TEST_PLAN.md`
- `docs/*.md`
- `baselines/*.md` and compact `baselines/*.json`
- `STUDY_EVIDENCE_INDEX.md`
- `experiments/*.py`, `experiments/README.md`, `experiments/EVIDENCE_SCHEMA.md`
- `coral_reef_spinnaker/` source and runtime source under `spinnaker_runtime/`
- compact generated registry/table/audit files needed to reproduce the public evidence index

## Evidence Kept In Git

Commit compact, claim-bearing evidence artifacts such as:

- tier reports: `*_report.md`
- results manifests: `*_results.json`, `*_latest_manifest.json`
- summary tables: `*_summary.csv`, compact comparison CSVs
- generated registry/table/audit files
- selected small figures or time-series files that are explicitly listed as
  expected artifacts in the evidence registry
- frozen baseline locks and registry snapshots

If a file is needed by `experiments/evidence_registry.py` as an expected artifact,
it may remain in Git even if it is generated. The key distinction is that it is a
curated evidence artifact, not raw intake.

## Artifacts Not Kept In Git By Default

Do not commit these as ordinary source:

- raw EBRAINS/JobManager returned downloads
- `reports.zip`
- SpiNNaker provenance databases such as `global_provenance.sqlite3`,
  `input_output_database.sqlite3`, `data.sqlite3`, or `ds.sqlite3`
- raw `spinnaker_reports/`, `raw_download/`, `raw_reports/`,
  `raw_hardware_artifacts/`, `downloaded_files/`, and `_download_intake_originals/`
- stack traces and status markers such as `stack_trace`, `errored`, and `finished`
- generated EBRAINS upload bundles nested inside result directories
- compiled `.elf`, `.aplx`, `.o`, host test binaries, and build directories
- local caches, scratch roots, and private AI handoff/audit notes

These files may be preserved locally or archived externally. If they are needed
for a claim, cite them through a manifest/hash/release rather than placing the raw
blob in the source tree by default.

## Evidence Categories

| Category | Meaning |
| --- | --- |
| Canonical registry evidence | Entries in `controlled_test_output/STUDY_REGISTRY.json` that populate the paper-facing table. |
| Baseline-frozen evidence | A mechanism/runtime state that passed its gate and has a frozen lock under `baselines/`. |
| Noncanonical diagnostic evidence | Useful evidence that answers a design question but does not freeze a new claim. |
| Failed/parked evidence | Negative evidence retained to explain why a path was not promoted. |
| Hardware prepare/probe evidence | Operational package/probe output; not a claim until returned artifacts are ingested and documented. |

## Generated Files

The canonical evidence ledger is generated by:

```bash
python3 experiments/evidence_registry.py
```

It writes:

- `controlled_test_output/STUDY_REGISTRY.json`
- `controlled_test_output/STUDY_REGISTRY.csv`
- `controlled_test_output/README.md`
- `STUDY_EVIDENCE_INDEX.md`

The paper-facing results table is generated by:

```bash
python3 experiments/export_paper_results_table.py
```

It writes:

- `docs/PAPER_RESULTS_TABLE.md`
- `controlled_test_output/PAPER_RESULTS_TABLE.csv`

The public repository audit is generated by:

```bash
python3 experiments/repo_audit.py
```

It writes:

- `docs/RESEARCH_GRADE_AUDIT.md`
- `controlled_test_output/RESEARCH_GRADE_AUDIT.json`

## EBRAINS Intake Rule

A file downloaded from EBRAINS or from a local Downloads folder is not canonical
by itself. It becomes evidence only after the relevant ingest path copies the
compact artifacts into `controlled_test_output/`, verifies pass/fail criteria,
records the claim boundary, and updates the docs/registry where required.

## Public Repository Rule

A clean public repository should let a reviewer answer four questions quickly:

1. What exactly is CRA?
2. What has been shown?
3. What has not been shown?
4. Where are the artifacts and pass/fail criteria?

If an artifact does not help answer those questions, it should usually be ignored,
externalized, or summarized.
