# Repo Alignment Remediation Plan

Last updated: 2026-05-15.

This document is the active cleanup checklist for restoring the public Apache
2.0 repository to research-grade, source-of-truth alignment after the
post-4.32 / post-7.7 mechanism exploration burst.

## Purpose

The cleanup goal is not to erase exploratory work. The goal is to separate:

- promoted baselines from diagnostic snapshots,
- canonical evidence from noncanonical or fallback-contaminated evidence,
- opt-in research mechanisms from default organism behavior,
- public claims from internal hypotheses.

No new science run should be started until this checklist is closed or an
explicit exception is documented.

## Current Audit Findings

1. Source-of-truth drift exists between `README.md`, `codebasecontract.md`,
   `docs/MASTER_EXECUTION_PLAN.md`, and baseline files.
2. `v2.7` is over-framed if it is described as superseding `v2.6` for
   predictive usefulness. It is a diagnostic NEST organism-development
   snapshot unless a later promotion gate proves otherwise.
3. Several mechanisms labeled experimental defaulted to enabled. Experimental
   mechanisms must be opt-in unless a clean promotion gate explicitly promotes
   them.
4. Some high-PR lifecycle/operator-diversity results were contaminated by NEST
   synthetic fallback and must not be used as promotion evidence.
5. Some mechanism implementations had code hygiene issues: duplicate config
   fields, duplicate readout update methods, and incomplete inhibitory scaling.
6. Several Tier 5.25+ artifacts and scripts were untracked while being
   discussed in source-of-truth documents.

## Evidence Tracking Decision - 2026-05-14

The Tier 5.24a and Tier 5.25+ mechanism artifacts/scripts currently under
review are small enough to keep in the public repository as audit history. The
decision is:

- Track the small cited diagnostic artifacts and the scripts that generated
  them instead of deleting or hiding them.
- Treat them as noncanonical or diagnostic unless they are explicitly added to
  `experiments/evidence_registry.py` and regenerated by `make validate`.
- Do not use fallback-era or partial-pass outputs as promotion evidence.
- Do not treat `*_latest_manifest.json` pointers from unregistered tiers as a
  source of truth; the generated registry remains authoritative.
- If a diagnostic result later becomes important enough for a paper claim,
  write a fresh promotion or registry gate instead of relabeling the old
  exploratory run.
- Do not commit large raw dumps, EBRAINS temporary folders, caches, or local
  downloads. Large artifacts need an explicit artifact-storage decision before
  they enter the public repo.

## Remediation Gates

### Gate 1 - Code Hygiene

Acceptance criteria:

- No duplicate `LifecycleConfig` mechanism fields.
- Experimental mechanism defaults are opt-in unless promoted.
- Getter fallbacks for experimental mechanisms are fail-closed.
- Duplicate readout update methods are resolved.
- Operator-diversity inhibitory scaling is actually applied.
- Unit tests pass.

### Gate 2 - Baseline / Claim Alignment

Acceptance criteria:

- `v2.6` remains the predictive benchmark baseline unless superseded by a clean
  promotion gate.
- `v2.7` is described as a diagnostic organism-development snapshot, not a
  predictive-performance supersession.
- Fallback-era PR claims are explicitly marked contaminated/noncanonical for
  promotion.
- `README.md`, `codebasecontract.md`, `docs/MASTER_EXECUTION_PLAN.md`,
  `docs/MECHANISM_STATUS.md`, and baseline files agree.

### Gate 3 - Evidence Registry Hygiene

Acceptance criteria:

- Every artifact referenced by source-of-truth documents is tracked, generated,
  or explicitly marked noncanonical.
- Partial/fail/diagnostic tiers cannot appear as promotion evidence.
- Generated registries and paper tables are refreshed after source changes.

### Gate 4 - Governance Tests

Acceptance criteria:

- Tests fail if an unpromoted experimental lifecycle flag defaults to `True`.
- Tests fail if `v2.7` claims to supersede `v2.6` for predictive usefulness.
- Tests fail if duplicate known-danger methods/fields return.
- Tests fail if fallback-contaminated text is used as promotion language.

### Gate 5 - Clean Rebaseline Decision

Status: contract locked by Tier 5.45 on 2026-05-15. The Tier 5.45a scoring
runner is implemented and smoke-validated, and the resumable shard orchestrator
is committed. The first four canonical shard cells
`organism_defaults_experimental_off` on sine seeds 42, 43, and 44 plus
Mackey-Glass seed 42 passed 10/10 with zero fallback, zero `sim.run`
failures, and zero summary-read failures. The full
Tier 5.45a scoring run remains the next required gate before any new mechanism
promotion, baseline freeze, or paper-facing claim.

Acceptance criteria:

- Healthy-NEST scoring reruns compare:
  `v2.6` predictive reference, organism baseline with experimental features
  disabled, each candidate mechanism opt-in, and the full opt-in stack.
- Required metrics include MSE, participation ratio, rank, sham separation,
  fallback counters, seed variance, and external baselines.
- Only mechanisms that improve task performance or robustly improve an
  explicitly predeclared substrate metric with sham separation can be promoted.

## Nonclaims During Cleanup

This cleanup does not prove a new mechanism, freeze a new software baseline,
claim external-baseline superiority, claim hardware transfer, or alter the
validity of earlier canonical hardware results.
